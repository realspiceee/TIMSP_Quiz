## Что реализовано
- Два режима: **Администратор** и **Тестируемый**.
- Сущности: `Question`, `Category`, `Administrator`, `Tested`, `MainMenu`.
- Хранение категорий и вопросов в `categories.json`.
- В репозитории уже подготовлен стартовый `categories.json` с 3 категориями: **Sports**, **Mathematics**, **Fruits** (по 10 вопросов в каждой).
- Выгрузка результатов тестируемого в `results.json`.
- Учет стоимости вопроса (баллы за каждый вопрос).
- Встроенные проверки/тесты граничных случаев через пункт меню `Запустить модульные тесты`.

## Архитектура файлов
- `main.cpp` — точка входа, связывание модулей.
- `MainMenu.h/.cpp` — центральная логика, меню, загрузка/сохранение JSON.
- `Administrator.h/.cpp` — добавление/удаление/редактирование категорий.
- `Tested.h/.cpp` — прохождение тестов, подсчет и экспорт результатов.
- `Category.h/.cpp` — хранение вопросов, запуск категории.
- `Question.h/.cpp` — модель вопроса и редактирование.

## Сборка в Visual Studio 2019/2022/2026
1. Создайте **Console App** (пустой проект).
2. Добавьте все `.cpp` и `.h` файлы из этого набора в проект.
3. Установите стандарт: **C++17** (`/std:c++17`).
4. Соберите проект (Debug/Release).

## Запуск
1. При старте автоматически выполняется загрузка `categories.json` (если файл есть)(файл должен быть в папке проекта).
2. В `Main Menu` выберите:
   - `1` — логин в роль администратора или тестируемого.
   - `2` — показать категории.
   - `3` — запустить встроенные тесты граничных случаев.
3. При выходе категории автоматически сохраняются в `categories.json`.

## Формат `categories.json`
Программа использует простой JSON-формат:
```json
{
  "categories": [
    {
      "name": "Математика",
      "questions": [
        { "text": "2+2?", "answers": ["3", "4", "5"], "correct": 1, "points": 10 }
      ]
    }
  ]
}
```
Где `correct` — индекс правильного ответа, начиная с `0`.
